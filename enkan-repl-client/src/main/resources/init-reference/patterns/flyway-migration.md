# Flyway migration pattern

Flyway runs automatically when `FlywayMigration` component starts. It scans
`classpath:db/migration` and applies any files named `V<version>__<desc>.sql`
in ascending version order. You must generate at least one file or the schema
will be empty and queries will fail.

## File layout

```
src/
  main/
    resources/
      db/
        migration/
          V1__create_todo_table.sql     ← this file
```

## Filename rules

- **Prefix `V`** (uppercase), then the version number.
- **Two underscores** `__` between version and description.
- Description uses `snake_case` — spaces are not allowed.
- Extension `.sql`.
- Examples: `V1__initial.sql`, `V2__add_author.sql`, `V3__drop_legacy.sql`.

## H2-compatible DDL template

```sql
-- V1__create_todo_table.sql
CREATE TABLE IF NOT EXISTS todos (
    id        BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    title     VARCHAR(255) NOT NULL,
    completed BOOLEAN NOT NULL DEFAULT FALSE
);

-- Optional seed data (keep this in a separate V2 file when possible):
INSERT INTO todos (title, completed) VALUES ('Example', FALSE);
```

## H2 type cheatsheet

| Java / Raoh decoder | H2 column type | Nullable form |
|---|---|---|
| `long_()` → `Long` | `BIGINT` | `BIGINT NULL` |
| `int_()` → `Integer` | `INT` | `INT NULL` |
| `string()` → `String` | `VARCHAR(n)` | `VARCHAR(n) NULL` |
| `bool()` → `Boolean` | `BOOLEAN` | `BOOLEAN NULL` |
| `decimal()` → `BigDecimal` | `DECIMAL(p, s)` | `DECIMAL(p, s) NULL` |
| `date()` → `LocalDate` | `DATE` | `DATE NULL` |
| `dateTime()` → `LocalDateTime` | `TIMESTAMP` | `TIMESTAMP NULL` |

## Auto-increment primary key

Prefer `GENERATED BY DEFAULT AS IDENTITY` over the older H2-specific
`IDENTITY` type:

```sql
id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY
```

This plays well with jOOQ's `.returning(field("id")).fetchOne()`.

## Do NOT

- Do not use MySQL / PostgreSQL-specific syntax (`AUTO_INCREMENT`, `SERIAL`,
  `CREATE TABLE IF NOT EXISTS ... ENGINE=InnoDB`, `TEXT` without size, etc.).
  The default stack is H2.
- Do not place migrations outside `src/main/resources/db/migration`.
- Do not use filenames like `v1_init.sql` (lowercase `v`, single underscore) —
  Flyway will ignore them.
- Do not leave the directory empty. Flyway's classpath scan finds nothing and
  silently skips the migration step, resulting in missing tables at query time.
